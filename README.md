# h5-portal-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面彼此独立，不依赖于特定的域名或后端服务，适用于快速展示、测试或作为静态资源参考。

## 项目简介

随着前端开发中 H5 页面的增多，有时需要在一个统一的位置管理多个独立页面，以便于分享、演示或归档。`h5-portal-guide` 正是为此而生。它是一个轻量级的页面集合仓库，每个页面都是一个完整的 HTML 文件，可以直接在浏览器中打开使用。

## 目录说明

```
h5-portal-guide/
├── pages/            # 存放所有独立 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/           # 公共资源文件（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html        # 入口页面（可选），用于导航或索引
└── README.md         # 本文件
```

- **pages/**：每个 HTML 文件代表一个独立的页面，可直接访问。
- **assets/**：存放页面共用的静态资源，便于复用和维护。
- **index.html**：可选的索引页面，用于快速跳转到各个页面。

## 页面归档说明

- 每个页面应是一个完整的、自包含的 HTML 文件。
- 页面间无依赖关系，可独立运行。
- 建议在 `pages/` 目录下按功能或项目分类建立子目录（可选），以保持结构清晰。
- 若页面需要引用资源，请优先使用相对路径，并确保资源位于 `assets/` 目录下。

## 维护说明

- 新增页面：在 `pages/` 下创建 HTML 文件，并更新 `index.html`（如有）中的导航链接。
- 修改页面：直接编辑对应文件，注意保持页面独立性。
- 删除页面：移除文件及对应的导航链接。
- 公共资源：如需增加全局样式或脚本，请放入 `assets/` 对应子目录，并确保页面正确引用。

## 使用方式

1. 克隆本仓库到本地。
2. 在浏览器中打开 `pages/` 下的任意 HTML 文件即可查看对应页面。
3. 若存在 `index.html`，可先打开该文件，通过导航链接访问其他页面。
